notes for the dev
– four scenes, one stack: same color tokens, same status-bar pattern, same `┌─ PROBE ─┐` header. Switch with the chip above; in the real TUI these are routes.
– status line is always the bottom row, full width: NORMAL mode pill (inverse-video amber) → contextual text → right-side keymap or counters.
– box-drawing uses U+2500 series throughout; selected sidebar/list rows use inverse-video amber (#d9a548 bg, dark fg). Soft-selected rows use a 14% amber wash.
– stage status colors match the GUI exactly: ● fresh / ● stale / ● running / ○ queued. Don't drift these between surfaces.
– the project scene shows both tabs (stages above, gantt timeline inline below) so you can see the relationship at a glance — in the actual TUI these are ⇧1 / ⇧2 alternates of the main pane, not stacked.
– gantt: bars are █ (solid = fresh), ▒ (dashed = pending rerun), ░ (light = queued/not-started). Today and deadline are vertical guides; stale stages are amber.
– every scene supports a :e command to drop into $EDITOR on the underlying TOML or run JSON; NO_COLOR falls back to monochrome.
