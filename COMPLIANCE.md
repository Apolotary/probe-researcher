# Hackathon scope — what was built April 21–26, 2026

This file documents which directories and files in the repo are part
of the submitted artifact and which are pre-existing context. The
`README.md` keeps a one-sentence pointer here so the lead reads as
"what we built", not as a long disclaimer.

The hackathon is the [Cerebral Valley *Built with Opus 4.7*
hackathon](https://cerebralvalley.ai/e/built-with-4-7-hackathon),
April 21–26 2026. Solo submission by Bektur Ryskeldiev.

## Built during the hackathon (the submitted artifact)

The submitted work is the interactive web/TUI workflow + the live
Anthropic web pipeline + the v2 Opus features (audit card, RQ
synthesis, replay infrastructure, provenance guard, side-by-side
Sonnet/Opus comparison).

### Source files in scope

- `src/cli/ui_app.tsx`, `src/cli/ui_scenes/`, `src/cli/ui_state.ts` —
  TUI router + scenes + workflow state for the new interactive
  command.
- `src/llm/probe_calls.ts` — async functions that hit the Anthropic
  SDK directly for the web/TUI workflow. **v2 additions:**
  `disagreementAudit` (forced-contrast schema), `rqBoolean` (RQ
  synthesis), `rebut` (functional rebuttal handler).
- `src/web/server.ts`, `src/web/probe_api.ts` — Express server +
  `/api/probe/<stage>` endpoints. **v2 additions:** rate limits (30
  req/min/IP), provenance guard for report / findings / review
  payloads, `/api/probe/models` for per-stage routing,
  `/api/probe/config` for live TOML round-trip,
  `/api/probe/audit-compare` for the Sonnet vs Opus side-by-side,
  `/api/probe/rebut` for the functional rebuttal stage.
- `src/web/probe_design/` — the design-handoff JSX + HTML, heavily
  extended. **v2 additions:** Opus-amber model badges, RQ-synthesis
  UI on brainstorm, replay-active pill, real recents from disk, the
  `<DisagreementAuditCard>` and `<SonnetCompareCard>` on the review
  screen, the `<TaggedView>` per-paragraph provenance UI, the
  `<PlaceholderTag>` "live · model output" / "placeholder" pill, the
  v3 readability rebuild (5-step gray ramp + Inter Tight prose +
  68ch measure cap).
- `src/web/probe_demo.ts` + `assets/demos/focus-rituals.json` — save
  / replay infrastructure with the bundled gold demo (~62KB, all 9
  stages, 3 reviewers, full audit). Replay is deterministic re-run
  of cached payloads.
- `assets/demos/focus-rituals-sonnet-audit.json` — Sonnet 4.6 audit
  of the same review panel as the bundled Opus audit, generated by
  `scripts/regen_audit_sonnet.ts`. Surfaces in the UI as the side-
  by-side comparison.
- `src/config/probe_toml.ts` — atomic config layer with per-stage
  model resolver, 0600 file permissions, redaction of stored keys
  on read.
- `scripts/regen_demo_v2.ts` — regenerates the bundled demo with
  real Opus 4.7 calls (audit + RQ synthesis slices). ~$0.10 per run.
- `scripts/regen_audit_sonnet.ts` — regenerates the Sonnet 4.6 audit
  comparison file. ~$0.05 per run.
- `scripts/e2e_smoke.sh` — 55-step end-to-end smoke harness for the
  web UI. Exercises every route, every stage endpoint, and the
  replay flow.
- `scripts/walk-all-screens.mjs` — headless-Chrome screenshot walker
  that produces the 17-PNG `docs/screenshots/all-screens/` set.
- `tests/` — vitest suite, 97 tests passing.
- `docs/screenshots/all-screens/` — the 17-PNG walkthrough set used
  by the readability audit and as a judge reference.

## Pre-existing reference material (not part of the submission)

The repository also contains an older, non-interactive offline
pipeline behind the `probe run` command. It exists primarily as
rigor context and a reference engine, not as the submitted artifact.

- `src/anthropic/client.ts` — older Anthropic client wrapper used
  only by the offline pipeline.
- `src/lint/` — provenance + forbidden-phrase linters that operate
  against shipped guidebook markdown. The web pipeline ports the
  forbidden-phrase regex set into a server-side guard
  (`src/web/probe_api.ts`'s `served()` helper) but the lint command
  itself is older.
- `src/orchestrator/` — branch dispatch + worktree management for
  the offline pipeline.
- `src/render/` — markdown rendering for the offline guidebook
  output format.
- `runs/` — 19 benchmark research runs from earlier work,
  documenting how the offline pipeline behaves at scale. Kept in
  the repo as a rigor reference rather than as submitted output.
- `corpus/source_cards/` — 12 hand-curated HCI source cards. The
  offline pipeline grounds citations against these; the web pipeline
  does not currently reference them.
- `patterns/` — 16-pattern capture-risk library for the offline
  pipeline.
- `agents/<role>.md` — agent system-prompt files for the offline
  pipeline's stages.

The older offline pipeline shipped in earlier internal iterations of
this project; the v2 web/TUI workflow is what the hackathon is
about.

## Verification

If a judge wants to verify the boundary:

```bash

git log --since=2026-04-21 --until=2026-04-27 --name-only --pretty=format: | sort -u

# All v2 commits (the submitted features):
git log --grep='v2\|disagreement\|audit\|rebut\|provenance\|rq.synthesis' --oneline

# Test suite + e2e smoke:
npm test          # 97 / 97
bash scripts/e2e_smoke.sh   # 55 PASS / 0 FAIL
```

The bundled `assets/demos/focus-rituals.json` is checked-in cached
state; the cache itself is the artifact, not the script that
generated it. Both `regen_demo_v2.ts` and `regen_audit_sonnet.ts`
will reproduce the cache from real Anthropic calls if a judge wants
to verify with their own API key.
